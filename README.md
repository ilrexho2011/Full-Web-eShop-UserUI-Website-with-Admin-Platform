# Full-Web-eShop-UserUI-Website-with-Admin-Platform

Automate the creation of sales orders of „CompuMac“ – Bodensee Sales System, JS, using SAP Integration Suite, UI5 APIs, and S4/Hana DB)
- iFlow creation in SAP Cloud Integration 
- Create a web API in SAP Fiori (UI5) to start the process,
- Create & Publish an Event in ERP Sytem
- Distribute the event through AEM broker to all Consumers (SAP CI, POS & S4/Hana
- Return the acknowledgement (Ack) event after processing the message
- Using Cloud Connector to manipulate the S4/Hana database
- CRUD operations through Odata adapter in SAP S/4Hana database
- Frontend development with PHP, HTML5 & CSS
- Server backend development with JS and Node.js)

Key requirements and indicators of the eShop Web Application.

Note: This document is only a preview platform of the eShop web application and serves as the starting point for the approval of module creation. Subsequently, after the approval of a quote, the platform will be detailed by becoming the complete documentation of the final application.

Technologies to be used:
Php, Javascript, SAP UI5, SAP CI, API Management, SAP AEM, SAP S4/Hana, SQL, Html5, CSS, Json, Framework: CodeIgniter 

Basic System requests for the installation of the eShop Web Application:
Linux Server, Php 8.x, Php myAdmin, MySQL

Expected objectives of the application:
-	Bootstrap User Frontend UI (German and English)

The user can register (verification of his authenticity can be done by sms, whatsapp or email), consults the products arranged in categories, using different filters for this purpose, can go live with the support provided, can contact at any time by email or voice chat, performs online purchases with different transaction methods, creates tickets for online support, but can also set physical appointments for professional support, is constantly informed about the status of various actions and processes related to it, etc.

-	Backend Administration Panel (German and English)

Admin Roles: Super Admin, Admin, Editor and Supporter

Below we list a series of actions which can be attributed to the above roles: 

Dashboard that provides a series of information and various statistical data, Creating new article categories and editing or deleting existing categories, Adding or editing Brand items (if necessary),
Adding new Products registered for sale as well as editing existing ones,Creating and publishing campaigns with "Flash Sales" with limited time limits,Creating and publishing offers with "Seasonal Discount" of the price by percentage,Adding and managing "Promo Codes" for the most privileged customers, Stock management of various items, management of sales invoices, moment inventory for the entire activity,Administration of live Chat conversations with customers, sending various messages by email or SMS to customers, for certain notifications,Tracking and management of Tickets opened by different customers,Full maintenance of customer list,  their data, addresses, transactions performed, etc.
Management of the return of items (in case it will be used),
Provision and management of "Local delivery" shipments,Management of all indicators and parameters of the system such as: 
- Authenticity and security of dates in the System,- Payment methods and their configuration,
- Shipping and Return Policy,
- Privacy Policy, Admin Policy, FAQ,
- Generation and management of API Keys, - System Updates- Management of themes, language and other system settings.
